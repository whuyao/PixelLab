# GeoAI Pixel Lab 系统发展过程复盘

副标题：从像素实验室原型到多智能体社会经济世界

## 1. 文档目的

这份文档不是单纯记录“系统现在有什么功能”，而是复盘：

- 这个系统最早从什么问题出发
- 它是如何一步步从聊天原型长成持续运行世界的
- 哪些关键升级改变了系统的性质
- 为什么今天它已经不再只是一个前端 demo，而是一个可研究、可讲解、可长期观察的多智能体系统

适用场景：

- 项目整体汇报
- 给新加入成员快速建立全局理解
- 面向本科生 / 研究生解释系统演化
- 写论文或技术报告时梳理系统设计脉络

本文默认对应当前主分支可运行系统；截至本文更新时，当前世界状态版本为 `60`。

## 2. 一句话概括整个发展过程

`GeoAI Pixel Lab` 的发展过程，可以概括成：

`从“实验室里几个人会说话”逐步演化成“一个带空间、社会、市场、财政、旅游和公共服务的持续运转世界”。`

也就是说，系统的复杂性不是一次性设计出来的，而是在多轮迭代中，把新的制度层和行为层不断接进旧世界后形成的。

## 3. 发展主线

整个系统的发展，可以分成七条并行主线：

1. `空间主线`
- 从简单实验室地图，发展到可缩放、可拖拽、可点击的田园像素世界
- 后续又进一步发展到分层地表、专属角色形象和地图事件反应层

2. `角色主线`
- 从固定 NPC，发展成带欲望、记忆、关系、信用和生活状态的智能体

3. `交互主线`
- 从按键触发对话，发展成玩家输入、观察模式、自动发言、自动交易和宏观干预

4. `经济主线`
- 从基础现金和简单交易，发展成股票、借贷、存款、消费、地产、工资、通胀和资产配置

5. `制度主线`
- 从没有公共规则，发展成税制、监管、财政结算、保障和公共投资

6. `外部性主线`
- 从只有实验室内部互动，发展成新闻、系统事件、游客、外来消息和节庆活动

7. `可观测性主线`
- 从看结果，发展成日志、晨报、分析图、经济事件流和结构化文档

8. `界面组织主线`
- 从单页堆叠，发展成多视图驾驶舱、市场台、企业竞争页、游客观察页、生活与住房页、财政页、小镇微博页、日志观察台、晨报与案件页和专门的教学页
- 从“页面展示事件”发展成“事件也能在地图上被看见”

## 4. 第一阶段：实验室原型期

最初的系统更像一个“会动的聊天实验室”。

这一阶段的核心组成是：

- 一张实验室地图
- 几个固定角色
- 玩家移动
- 靠近后触发对话
- 基础任务和时间推进

这一阶段解决的主要是：

- 能不能把“智能体”放到同一张地图里
- 能不能让玩家在空间上接近某个角色后再发生互动
- 能不能把静态对话变成“位置相关对话”

如果用研究语言来说，这阶段的重点是：

`先建立一个最小的空间化多角色交互环境。`

这时系统的本质还不是“社会模拟”，而是“带地图的交互原型”。

## 5. 第二阶段：关系与记忆期

系统进入第二阶段后，角色不再只是会说一句话，而开始有：

- 短期记忆
- 长期记忆
- `memory_stream`
- 当前活动
- 当前气泡
- 关系值
- 人格和说话风格

这一阶段最重要的变化是：

`角色开始具有“连续性”。`

这意味着角色不是每次被点击时重新生成一次回复，而是带着：

- 刚刚发生了什么
- 之前和谁关系变好了或变差了
- 自己当下最在意的事情是什么

去进入下一轮行为。

从系统性质上说，这一步非常关键，因为它让角色从：

- “会回答的问题对象”

变成了：

- “会带着内部状态生活的人”

## 6. 第三阶段：欲望驱动与社会线程期

再往后，系统不再只靠“人设标签”驱动对话，而是开始围绕欲望运行。

典型欲望包括：

- 恢复体力
- 缓解钱压
- 抓机会
- 守住边界
- 被接住
- 照顾别人
- 把话讲清
- 证明自己

同时，系统加入了：

- 联盟
- 冲突
- 调停
- 和解链
- 社会线程
- 故事线

这一步的重要意义是：

`角色不再只是“不同性格”，而是开始表现出不同的行为策略。`

例如：

- 有人更容易合作
- 有人更容易起冲突
- 有人会主动去接住别人
- 有人会更看重节奏和资源

这就是从“人物设定”走向“社会动态”的阶段。

## 7. 第四阶段：市场与金融期

系统最重要的一次性质跃迁，来自市场和金融系统的加入。

这一阶段逐步加入了：

- 股票市场
- 板块轮动
- 从 `3` 只教学股票扩到 `8` 只市场股票
- `PX100 / PX-Tour / PX-Life` 指数
- 牛市 / 震荡市 / 风险市
- 玩家交易
- 智能体自动交易
- 游客轻量投资
- 银行借贷
- 人际借贷
- 信用值
- 银行存款
- 动态利率
- 地下赌场
- 赌税
- 赌场热度

这一步的意义在于：

`系统第一次拥有了可持续的经济压力与经济机会。`

角色开始需要面对：

- 手头现金不足
- 信用下降
- 借款到期
- 市场波动
- 资产配置

于是行为不再只是“说什么”，而变成了：

- 是不是去借钱
- 要不要存款
- 什么时候买股票
- 什么时候卖资产
- 什么时候转向打工

从研究角度，这一步使系统从“社会互动模拟”转向了“社会经济互动模拟”。

赌场的加入很关键，因为它把高波动、高情绪、灰色边缘的经济行为接进了系统。和股票不同，赌场不是理性配置资产，而是更像社会边缘现金流节点：

- 有人会想赌一把翻身
- 有人有钱后会去试运气
- 游客会把赌局当成后巷传闻带进微博
- 政府会从赌税、灰案和监管里重新看见它

这意味着系统开始同时拥有正式金融、日常消费、地产经济和非正式高波动灰色经济。

赌场再往后还经历了一次关键平衡调整。最初它更像一个会快速抽走现金的高波动洞口，但后续版本逐渐把它推向了另一种角色：

- 缩小自动下注规模
- 提高赌局结果对情绪、满意度和消费欲望的反馈
- 强化赌场与微博炫耀、游客八卦、灰案和监管抽查的耦合

这样之后，赌场不再主要作为“吸钱机器”存在，而更像一个把风险偏好公开化、把灰色行为舆论化的社会节点。

## 8. 第五阶段：生活消费与地产期

如果只有股票和借贷，经济仍然会显得过于抽象。

因此系统后来又接入了：

- 生活满意度
- 日常消费
- 消费券
- 房产
- 地产买卖
- 地产维护费
- 房产收益
- 住房品质

这一步的核心价值是：

`让金钱真正回到“生活”层面。`

钱不再只是用于交易，而是会影响：

- 住得好不好
- 活得舒不舒服
- 是否愿意消费
- 是否更容易进入高满意度状态

地产系统则进一步把经济和空间绑定起来：

- 资产不再只是数字
- 房屋、店铺、温室、出租屋开始在地图上成为具体对象

这使系统从“金融模拟”进一步走向“空间化经济模拟”。

## 9. 第六阶段：政府、税制与公共服务期

当财富、交易、灰色活动、游客和地产都进来后，系统自然需要一个更高层的调节机制。

于是加入了：

- 工资税
- 证券税
- 地产过户税
- 地产持有税
- 消费税
- 奢侈税
- 监管抽查
- 财政储备
- 低收入补助
- 破产救助
- 15 天财政结算
- 公共服务、旅游支持、住房支持
- 政府资产投资

这一步让系统发生了一个本质变化：

`世界不再只是由个人决策构成，还开始受到制度性力量约束。`

财政系统的加入，使系统第一次拥有了：

- 再分配
- 稳定器
- 公共投资
- 监管惩罚
- 对高财富和灰线行为的制度反馈

在很多复杂系统研究里，这都是从“行为系统”走向“制度系统”的关键分水岭。

## 10. 第七阶段：游客与开放经济期

后来接入的游客系统，让这个世界从一个封闭实验室，变成了一个更像开放小镇的系统。

游客系统带来了：

- 旅馆
- 集市
- 游客消费
- 游客消息
- 回头客
- 高消费客户
- 潜在购房者
- 淡季 / 旺季 / 活动日

在后续稳定化阶段，游客机制又进一步改成了“总池 `10`、活跃 `7`”的双层模式，让世界状态、地图呈现和高频行为循环不再死绑在同一个小上限上。与此同时，游客命名也从早期的“中文名 + 数字后缀”过渡到自然中文名优先分配，并对旧状态自动清洗，减少了明显的程序痕迹。

游客的意义，不只是“增加几个人”，而是：

`把外部需求带进了系统。`

这会连锁影响：

- 居民收入
- 政府收入
- 市场情绪
- 资产价值
- 地方消费热度
- 对话内容和记忆来源

更重要的是，游客收入后来被进一步细分成：

- `私人收入`
- `财政资产收入`
- `公共运营收入`

这意味着系统开始区分：

- 游客的钱到底进了私人账户
- 还是进入了政府持有资产
- 还是进入公共文旅运营池

这是一个非常关键的制度化步骤，因为它让“旅游收入”从一个模糊数字，变成了一个可追踪的分配结构。

## 11. 第八阶段：可观测性和分析期

系统越复杂，就越不能只靠“肉眼看地图”来理解它。

因此后期又加入了：

- `activity.jsonl` 行为日志
- SQLite 快照
- `Lab Daily` 晨报
- 经济事件流
- 对话过滤器
- 实时分析图
- 人物剖面
- 近 100 天与 300+ 天运行分析报告

这一步很重要，因为没有观测层，再复杂的系统也只是“会动”而已。

加入可观测性之后，系统才真正具备了：

- 教学价值
- 分析价值
- 研究价值
- 长时程对比价值

## 12. 第九阶段：多视图驾驶舱期

当模块越来越多之后，系统又出现了一个新的问题：

- 地图、市场、财政、日志、地产都挤在一个页面里
- 单页虽然信息全，但理解成本极高
- 用户很容易在“正在控制什么”与“正在观察什么”之间迷失

因此后期前端又经历了一次信息架构重构：

- 首页变成“驾驶舱”
- 市场成为独立金融工作台
- 财政与政府成为独立制度页面
- 小镇微博成为独立公开舆论页
- 日志与观察变成专门的观测页
- 角色详情改成透明浮窗

这一步虽然不是底层经济制度升级，但对系统的使用体验和教学解释价值很关键，因为它把：

- 结果
- 控制
- 历史
- 细节

从视觉上分开了。

同时，人物剖面也不再只展示“现金、信用、压力、满意”四个最粗指标，而开始补入：

- 总资产
- 存款
- 房产数量与估值
- 股票持仓市值
- 银行待还

这意味着系统后期不只是“能看状态”，而是开始具备更强的资产结构解释能力。

## 12.1 大政府机制期

在多视图稳定之后，系统又往前迈了一步：政府不再只是财政参数集合，而开始被塑造成一个可切换强度的制度智能体。

这一阶段的重要变化有：

- 新增 `大政府模式` 总开关
- 政府支持度从静态口碑改成 `每日匿名投票`
- 投票机制从硬阈值分类改成 `连续得分 + 概率抽样`
- 大政府会根据当天民意偏向，在收入、秩序和发展之间做更稳的微调
- 增加 5 个细权限：
  - 调税
  - 调息
  - 建设拆除
  - 收购出售
  - 价格干预
- 政府不再只在财政周期里被动结算，而会更主动地响应：
  - 游客承压
  - 住房供给
  - 公共资产净收益
  - 市场阶段
  - 微博舆情与政府已知信号

但这一阶段也暴露了一个非常典型的“制度系统接入空间系统”问题：

- 政府设施如果没有空间锚点和工期约束，就会在地图上看起来像“说建就建、说挪就挪”

后续因此又补了两套约束：

- 建设与拆除进入 `construction / demolishing` 周期
- 已建成的不动产锁定锚点，不再漂移

这一阶段的意义在于，它让系统首次具备了“制度调控不仅改参数，还改地图”的能力。

## 13. 为什么说系统发生了“性质变化”

系统的发展不是简单堆功能，而是出现了几次明显的性质变化。

### 13.1 从对话系统到世界系统

最开始，系统主要关心“角色怎么说话”。

后来，系统开始关心：

- 谁在什么地方
- 为什么在那里
- 说完之后会发生什么后果

这时它就从对话系统，变成了一个世界系统。

### 13.2 从角色系统到经济系统

当现金、借贷、交易、消费、通胀、地产都接入之后，角色行为不再只是由关系驱动，也被经济约束驱动。

这时系统变成了一个经济社会系统。

### 13.3 从经济系统到制度系统

当政府、税制、监管、保障、公共服务和政府投资加入后，系统不再只是个体互动，而出现了制度层。

这时它开始具备更强的“社会模拟”属性。

### 13.4 从封闭系统到开放系统

游客与外部消息的引入，使系统不再只由内部角色构成，而开始不断接收外部扰动。

这一步，使系统更接近真实世界。

### 13.5 从私下互动到公开舆论层

当 `小镇微博` 加入之后，系统又出现了一次很关键的性质变化。

之前角色主要通过：

- 私聊
- 环境对话
- 最近事件
- 晨报

来影响彼此。

而在微博层加入后，系统第一次拥有了公开信号市场：

- 个人体验会被公开化
- 游客和政府会公开回应
- 热帖会传播、转发、被围观
- 高热帖子会进入 `Lab Daily`
- 热帖会反过来影响市场、游客和地图上的局部行为

这意味着系统不再只是“行为系统”，而开始具备“舆论传播系统”的性质。

再往后，微博又出现了一次关键升级：

- 从“规则帖流”升级到“规则草稿 + LLM 精修”
- 从只有时间线升级到 `总览 / 热榜 / 传播`
- 从单条热帖升级到楼中楼、围观人数、转发链和地图围观反应

这一轮变化很重要，因为它说明微博不再只是一个展示模块，而是真正变成了公开信号市场。系统开始区分两类语言层：

- 私下对话：更看关系、记忆和即时欲望
- 公开表达：更看立场、传播性、情绪和被谁看见

这让系统里的“语言”第一次具备了制度差异：同一个角色，在私聊里和在公开舆论场里，会表现成两种不同的话语方式。

## 14. 现在这个系统已经是什么

截至当前版本，它已经不是：

- 单纯的前端 demo
- 单纯的聊天原型
- 单纯的科研任务演示

而是一个：

`带空间地图、角色记忆、关系博弈、市场波动、资产配置、税制财政、游客流量、公共服务和行为日志的多智能体社会经济仿真系统。`

如果用更简洁的话说：

`它已经是一个会自己运转、自己积累后果、自己长出结构的小世界。`

## 15. 当前系统的关键成熟标志

可以用下面这些“成熟标志”判断它已经走到了什么阶段。

### 行为层

- 角色不再只会说话，而会工作、消费、投资、存款、借贷、休息、调停和冲突

### 经济层

- 已经有现金、股票、存款、房产、贷款、税收、公共投资等多资产层

### 制度层

- 已经有财政、监管、保障、税率调整和政府资产运营
- 已经有银行监控台、动态授信和日级银行历史

### 外部性层

- 已经有自动主线新闻时间线、游客消息、系统新闻台 fallback 和节庆活动

### 公开信号层

- 已经有 `小镇微博`、热榜、传播链、楼中楼、主题/情绪筛选、阅读锁定和热帖地图反应

### 观察层

- 已经有晨报、日志、经济事件流、分析图和长期运行报告

## 16. 下一步仍然可以怎么发展

虽然系统已经很厚了，但仍有几条自然的继续演化方向：

1. `人口流动`
- 新人搬入、旧人离开、居民名册和长期人口结构

2. `更强公共服务系统`
- 医疗、教育、交通、住房保障的更细化反馈

3. `更强产业分化`
- 让游客、地产、GeoAI、农业链形成更清晰的多产业结构

4. `更丰富的政策实验`
- 调税、补贴、监管、公共投资的长期效果对比
- 自动新闻时间线和政策变量之间的联动实验

5. `更强的研究可复现性`
- 固定实验种子
- 批量跑多次仿真
- 输出对比图和统计摘要

## 17. 结论

这套系统的发展过程，最值得强调的不是“功能越来越多”，而是：

`每一轮新机制的接入，都会改变旧系统的行为逻辑。`

地图让交互有了空间性。  
记忆让角色有了连续性。  
欲望让行为有了驱动。  
市场让决策有了风险。  
地产让财富有了锚点。  
财政让世界有了制度。  
游客让系统变成开放经济。  
日志和分析，则让这一切终于变得可解释。

这正是 `GeoAI Pixel Lab` 从原型成长为复杂系统的全过程。

## 18. 最近一轮修正

最近一轮系统修正主要收了两个容易误导观察者的问题：

- `主线任务漂移`
  旧逻辑会在每次刷新时把当前团队资金重新写成任务起点，导致任务文案持续跳动、进度失真。现在主线任务改成固定 `start_value / goal_value` 区间，进度只按当前资金在该区间中的位置计算。
- `关系可视化不足`
  角色详情里新增了玩家与 5 个核心智能体的关系矩阵，用颜色区分 `暧昧 / 默契 / 熟悉 / 友好 / 普通 / 紧张`，让关系网络不再只靠零散文本阅读。
