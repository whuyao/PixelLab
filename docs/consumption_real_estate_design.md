# 消费、生活满意度与房地产系统设计

本文档给出 `PixelLab` 下一阶段的经济扩展设计，目标是在不破坏现有地图、市场中心和右侧交互侧栏结构的前提下，引入：

- 日常消费系统
- 生活满意度系统
- 房地产买卖 / 建造 / 出租系统
- 消费贷 / 房贷 / 抵押式借贷
- 地图中的地产可视化

---

## 当前实现状态（截至世界版本 36）

本文最初是设计稿，但其中大部分核心机制已经落地。当前已实现：

- `生活满意度 / 消费意愿 / 住房品质`
- 可消费物品目录
- 房产买入、卖出、贷款买入
- 房产日结：维护费、租金或经营收入
- 左侧独立 `生活与地产` 面板
- 统一 `finance_history` 经济事件流
- 公司打工、日开销、通胀压力
- 银行贷款、银行存款、动态利率
- 政府财政、公共服务、消费券与政府资产
- 游客消费、旅馆、集市与潜在购房者

因此，本文应理解为“设计原则 + 已实现机制 + 后续扩展方向”的混合文档，而不再只是纯设想。

---

## 1. 设计目标

当前系统已经具备：

- 股票市场
- 玩家和智能体现金
- 信用分
- 人际借贷
- 银行借贷
- 灰色交易与地下案件
- 角色关系、欲望、记忆、作息

但目前仍有两个明显问题：

1. 大家的消费欲望偏低  
2. 金融系统主要集中在股票和借贷，缺少“生活资产”与“居住升级”的长期循环

本轮扩展要解决的不是“再加一堆按钮”，而是让角色在生活层面有更自然的花钱理由，并让玩家有新的长期目标。

---

## 2. 核心思路

### 2.1 从“赚钱”扩展到“生活”

现有资金流主要围绕：

- 聊天
- 借贷
- 股票
- 灰色交易

新增系统后，资金流将多出第二条主线：

- 日常消费
- 居住改善
- 地产投资
- 租金收入
- 满意度回报

这样角色不再只为了“周转现金”而行动，而会为了更舒服的生活、更稳定的住处、更好的社交状态而花钱。

### 2.2 消费不是纯扣钱，而是换满意度

消费的核心回报不设为“立即赚钱”，而设为：

- `生活满意度`
- `心情`
- `体力恢复效率`
- `关系提升机会`
- `口碑或社交吸引力`

这样消费就会变成角色欲望系统的一部分，而不是单纯成本。

### 2.3 房地产和消费独立成新面板

现有左侧主列已经有：

- 地图主舞台
- 市场中心
- 实验室主面板
- 当前角色信息

建议新增一个独立面板：

- `生活与地产`

它放在左侧主列，位于：

- `市场中心` 下方
- `实验室主面板` 上方

这样做的好处：

- 股票和地产不混在一个面板里
- 不会破坏右侧任务 / 对话 / 事件流
- 金融系统内部仍然分区清楚

## 3. 新系统总览

### 3.1 新指标

为玩家和智能体新增：

- `life_satisfaction`：生活满意度，范围 `0-100`
- `consumption_desire`：消费欲望，范围 `0-100`
- `housing_quality`：当前居住品质，范围 `0-100`
- `monthly_burden`：固定负担，含租金、维护、分期、房贷
- `materialism`：物质偏好，影响消费倾向
- `comfort_preference`：舒适偏好，影响住房升级欲望

其中：

- `life_satisfaction` 是结果指标
- `consumption_desire` 是行为驱动
- `housing_quality` 是资产反馈

### 3.2 满意度的主要来源

正向来源：

- 买到想要的生活品
- 住得更舒服
- 和喜欢的人一起消费
- 房产带来稳定收入
- 体力稳定、债务压力低

负向来源：

- 现金过低
- 债务过高
- 住房拥挤或质量差
- 连续熬夜
- 社交关系差
- 想消费却一直买不起

### 3.3 新资产层

新增两大类资产：

1. `消费品`
- 咖啡
- 热饭
- 小礼物
- 花束
- 装饰摆件
- 工具
- 家具

2. `房地产`
- 小屋升级
- 农田
- 出租屋
- 小商铺
- 温室 / 花房
- 湖边休闲亭

## 4. 为什么现在大家消费欲望不高

当前系统里角色更在意：

- 现金安全
- 信用
- 借贷压力
- 股票
- 灰色事件

但缺少明确的“花钱回报”闭环，因此理性行为会自然偏向：

- 攒钱
- 借钱
- 投机
- 低消费

### 4.1 需要补上的机制

建议增加以下消费驱动：

1. 满意度不足时，会主动想花钱改善状态  
2. 不同人格有不同消费偏好  
3. 某些社交事件需要消费才能推进  
4. 住房质量会持续影响恢复和情绪  
5. 日常消费会影响关系质量

### 4.2 提高消费意愿的具体规则

建议把消费意愿改成下面这套：

```text
基础消费意愿
= 角色物质偏好
+ 当前满意度缺口
+ 当前压力
+ 当前关系需求
+ 当前天气/时段诱因
- 债务压力
- 现金过低带来的保守倾向
```

更具体地：

- `life_satisfaction < 45` 时，消费欲望明显上升
- `energy < 40` 时，更容易想买吃的或休闲恢复
- `stress > 65` 时，更容易产生情绪消费
- `cash < 20` 时，普通消费下降，但“低价恢复型消费”上升
- `cash < 10` 时，会优先借款或分期

## 5. 消费系统设计

### 5.1 消费品分类

#### 快消类

- 咖啡
- 简餐
- 小零食
- 雨伞
- 花束

特点：

- 价格低
- 立刻提高心情 / 满意度
- 容易触发对话和关系变化

#### 生活改善类

- 小台灯
- 椅子
- 书架
- 床品
- 装饰画
- 植物盆栽

特点：

- 价格中等
- 持续影响 `housing_quality`
- 持续影响满意度恢复

#### 功能工具类

- 农具
- 数据终端升级件
- 小型传感器
- 整理工具

特点：

- 既能提升生活满意度，也能对生产效率产生少量正反馈

### 5.2 消费行为触发

消费可以由三种方式触发：

1. 系统自动消费
- 智能体根据欲望自动决定购买

2. 对话触发消费
- 一起喝咖啡
- 送礼物
- 请吃饭
- 帮对方买东西

3. 玩家主动购买
- 玩家在前端手动下单
- 选择为自己购买或作为礼物送人

### 5.3 消费与关系

消费不只改钱，还会改社交：

- 请客：关系上升
- 送礼：满意度上升，关系更容易升
- 被拒绝的礼物：尴尬或关系下降
- 借钱消费：容易增加未来压力和冲突

## 6. 房地产系统设计

### 6.1 房产类型

建议第一版先只做 4 类：

1. `小屋升级`
- 改善居住品质
- 提高休息效率
- 提高满意度上限

2. `农田`
- 可稳定产出少量收入
- 也可作为装饰型地产，提升氛围

3. `出租屋`
- 稳定租金收入
- 但有维护成本、空置风险

4. `小商铺`
- 收益更高
- 更受口碑、天气、市场阶段影响

### 6.2 房产属性

每个房产建议有：

- `id`
- `owner_type`：`player / agent`
- `owner_id`
- `property_type`
- `name`
- `position`
- `purchase_price`
- `estimated_value`
- `daily_income`
- `daily_maintenance`
- `comfort_bonus`
- `social_bonus`
- `status`：`owned / listed / rented / mortgaged`

### 6.3 房产对系统的影响

房产不仅是收益品，还影响角色生活：

- 小屋升级提高休息恢复
- 农田提升田园氛围和满意度
- 出租屋带来现金，但会增加管理压力
- 商铺影响口碑和消费流

### 6.4 买入卖出与建造

玩家可进行：

- 买入已有房产
- 卖出持有房产
- 在空地上建造新地产
- 升级现有房产

智能体也可进行：

- 自动买小型资产
- 租房 / 合租
- 升级小屋
- 在高现金且高消费欲望时尝试投资地产

## 7. 贷款与分期系统扩展

### 7.1 新贷款类型

现有银行借贷可扩展成：

- `消费贷`
- `房贷`
- `装修贷`
- `短期分期`

### 7.2 利率规则

建议：

- 消费贷利率最高
- 房贷利率最低
- 装修贷居中
- 信用越低，利率越高
- 实验室口碑差时，银行整体收紧授信

### 7.3 不够钱时的购买逻辑

当角色现金不足：

1. 先判断购买是否刚需
2. 再判断是否可分期
3. 再判断是否可申请消费贷或房贷
4. 再判断是否放弃消费

这能自然解释：

- 为什么有人会冲动贷款买家具
- 为什么有人会先忍住不消费
- 为什么有人会为了房子承担更高利率

## 8. 新前端布局方案

### 8.1 原则

不打乱现有右侧侧栏，不把股票和房地产混成一锅。

### 8.2 左侧主列新顺序

建议变成：

1. 地图主舞台
2. 市场中心
3. `生活与地产`
4. 实验室主面板
5. 当前角色信息

### 8.3 新面板结构：生活与地产

#### 左列：生活消费

- 玩家可购买商品
- 今日消费记录
- 团队满意度分布
- 最近礼物 / 请客 / 共同消费

#### 右列：房地产

- 房产列表
- 房产状态卡
- 买入 / 卖出 / 升级按钮
- 贷款购买入口

### 8.4 可视化地图层

地图上的地产建议用下面方式呈现：

- 小屋升级：屋子更大、装饰更丰富
- 农田：扩大的耕地或花圃
- 出租屋：额外小屋或连排小房
- 商铺：有招牌的小建筑

点击房产可显示：

- 所有者
- 当前估值
- 日收益
- 维护费
- 住房舒适度加成

## 9. 后端模块拆分建议

当前 `GameEngine` 已经开始拆成：

- `MarketEngine`
- `SocialEngine`

消费与地产建议独立成第三个经济生活模块：

- `LifestyleEngine`

职责：

- 消费意愿计算
- 满意度更新
- 商品购买结算
- 房产估值
- 房贷 / 消费贷申请辅助
- 房产收益与维护费结算

这样可以避免继续把复杂规则堆回 `GameEngine`。

## 11. 系统联动规则

### 11.1 消费 -> 满意度

- 普通消费：`+2 ~ +6`
- 礼物型消费：双方关系变化
- 居住改善型消费：持续正向加成

### 11.2 房地产 -> 现金流

- 农田：低收益、低波动
- 出租屋：中收益、中维护
- 商铺：高收益、高口碑敏感

### 11.3 满意度 -> 行为

- 高满意度的人更稳定，关系更好，冲突更少
- 低满意度的人更容易：
  - 情绪化消费
  - 借钱
  - 与人起冲突
  - 做短视投资

### 11.4 地产与股市联动

可选二期规则：

- 房地产繁荣会拉动 `AGR`
- 商业地产热度会带动 `SIG`
- 实验室扩建或空间智能地产里程碑会利好 `GEO`

## 12. 推荐开发顺序

### 第一阶段：生活满意度 + 日常消费

目标：

- 先让大家“愿意花钱”
- 不先做复杂地产交易

交付：

- 新指标：满意度、消费欲望、住房质量
- 消费品清单
- 自动消费逻辑
- 玩家消费入口

### 第二阶段：生活与地产面板

目标：

- 把消费和地产从股票区独立出来

交付：

- 新前端模块 `生活与地产`
- 消费记录区
- 房产列表区

### 第三阶段：房产买卖与升级

目标：

- 玩家可以拥有和经营地产

交付：

- 房产数据模型
- 买入 / 卖出 / 升级
- 地图可视化

### 第四阶段：消费贷 / 房贷

目标：

- 让“大额消费”有金融杠杆

交付：

- 新贷款类型
- 利率规则
- 到期与违约结算

## 13. 当前建议结论

建议不要一次把“所有消费、所有地产、所有贷款、所有装修、所有租客系统”同时做完。

最合理的落地顺序是：

1. 先做 `满意度 + 消费意愿 + 基础消费`
2. 再加左侧独立面板 `生活与地产`
3. 再做房产资产和地图可视化
4. 最后接房贷 / 消费贷 / 分期

这样做的好处：

- 能最快解决“大家消费意愿不高”的核心问题
- 不会把现有市场中心搞乱
- 可以让房产系统长成一套完整的第二经济循环

---

## 14. 建议的 MVP

如果下一轮直接开工，建议先只做这个最小版本：

- 玩家和智能体新增 `生活满意度`
- 每人每天自动发生 1~3 次小消费
- 消费能提升满意度、心情和关系
- 前端新增左侧 `生活与地产` 面板
- 玩家可以买：
  - 咖啡
  - 礼物
  - 家具
- 玩家可以买 3 类地产：
  - 小屋升级
  - 农田
  - 出租屋
- 不够钱时可申请：
  - 消费贷
  - 房贷

这版已经足够形成新的可玩循环。
