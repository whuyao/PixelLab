# LLM Prompt 调优复盘

本文复盘这一轮围绕 `OpenAI / Qwen` 双模型兼容层做的语言收敛、提示词瘦身、口吻分流和长期记忆整理。目标很明确：

- 减少无意义的 thinking 和长输出
- 让 NPC 对话、微博原发帖、微博回复更像人话
- 让 5 个核心角色在私聊和公开场合说话明显不同
- 清掉 `data_wall` 这类内部代号泄露
- 让长期记忆从“重要短期记忆复制”变成“更少、更稳的长期挂念”

## 1. 触发问题

在稳定版之前，系统有 5 类明显问题：

1. 对话和微博太容易出现“分析腔”“客服腔”“汇报腔”。
2. 不同角色串味，林澈、米遥、周铖、芮宁、凯川会冒出相似句式。
3. 会出现人类不太会说的词：
   - `锚点`
   - `新口子`
   - `盯得更紧`
   - `编剧`
4. 微博回复常像“已读乱回”，接不上原帖的具体语境。
5. 长期记忆更像高重要度短期记忆的复制品，更新偏频，而且内容不够聚焦。

## 2. 调整目标

这轮不是简单“把 prompt 再写长一点”，而是做了四个方向的结构化收敛：

- 任务分流：对话 / 微博原发帖 / 微博回复分别用不同模板
- 模型分流：`Qwen` 和 `GPT` 分别做不同的收口
- 角色分流：核心 5 人拆成私聊人格和公开人格两套钩子
- 输出清洗：在模型输出后再做一层中文清洗和长度压缩

## 3. Prompt 结构调整

核心文件：

- [app/services/openai_dialogue_service.py](../app/services/openai_dialogue_service.py)

### 3.1 对话、原发帖、回复分开

不再共用一套“万能角色提示词”，而是拆成：

- `NPC 对话`
- `微博原发帖`
- `微博回复`
- `政府原发帖`
- `政府回复`

这样做的原因是：

- 私聊要像面对面接话
- 原发帖要像“忍不住发一句”
- 回复要像“先接住原帖，再明确表态”
- 政府发帖不能像网友吵架，也不能像空泛公文

### 3.2 Qwen 与 GPT 分开收

这轮没有追求“一套 prompt 通吃所有模型”，而是分别针对两类模型调：

- `Qwen`
  - 关闭 `thinking`
  - 强压“先别急 / 再看看 / 平衡分析”
  - 鼓励短句、生活说法、第一反应
- `GPT`
  - 压制“完整逻辑链 / 三段论 / 标准答案感”
  - 强调“先像人接话，再补半句”

## 4. 角色口吻分流

### 4.1 私聊人格钩子

5 个核心角色现在有单独的私聊口吻：

- `林澈`
  - 克制、拆事、先挑不对劲的地方
- `米遥`
  - 更软、更别扭、更容易露情绪
- `周铖`
  - 直接、落地、迅速落到“谁来干、谁来扛”
- `芮宁`
  - 先接情绪，再指出谁在硬撑
- `凯川`
  - 更像悄悄交换判断，谈风向与危险

### 4.2 公开人格钩子

微博公开表达则进一步区分成另一套：

- `林澈`
  - 证据、边界、代价、拆逻辑
- `米遥`
  - 敏感、跳跃、个人感受更强
- `周铖`
  - 直接、算账、谁背锅
- `芮宁`
  - 看谁会被压住、谁会先撑不住
- `凯川`
  - 风向、趋势、谁会被卷进去

这让角色出现一个更真实的分层：

- 私下说法和公开表态不再完全一样
- 公开人格不会简单复读私聊口气

## 5. 政府发帖单独模板

政府相关发帖和回复，已经完全脱离普通微博模板。

现在的要求是：

- 正式
- 短
- 稳
- 清楚
- 交代依据、边界和下一步

这使得：

- 政府原发帖更像公告
- 政府回复更像正式回应
- 不再像普通网友吵架

## 6. 输出后处理清洗

为了避免模型偶发吐出奇怪口头禅，这轮加了输出后处理。

典型替换包括：

- `别急着冲` -> `先稳一下`
- `先别急` -> `先缓一下`
- `再看看` -> `再想想`
- `顶着一口气` -> `谁都不想先退`
- `盯得更紧` -> `看得更紧`
- `新口子` -> `新路子`
- `锚点` -> `抓手`
- `编剧` -> `外面那帮人`

同时还会：

- 压短句子数
- 去重重复标点
- 裁掉微博和回复里过长的尾句

## 7. data_wall 问题与修复

用户后来发现智能体会说 `data wall`。

这不是世界设定的一部分，而是内部地图代号，真实地名是：

- `data_wall` -> `果园坡地`

根因是提示词里把 `agent.current_location` 原样喂给了模型。现在已经改成只给中文地名，不再暴露内部房间键。

## 8. 长期记忆的收口

核心文件：

- [app/engine/game_engine.py](../app/engine/game_engine.py)
- [static/app.js](../static/app.js)

之前的长期记忆机制本质上是：

- 短期记忆写入
- 某些事件带 `long_term=True`
- 原文直接复制到 `long_term_memory`

所以会出现：

- 重点不够聚焦
- 同一天刷多条
- 连续几天挂三条相似市场记忆

现在改成：

- 长期记忆先摘要成“长期挂念”
- 同一天默认只进 1 条，特别重要才允许到 2 条
- 同签名的内容不重复堆入
- `Memory Stream` 只在跨天或重要度更高时展示长期挂念
- 连续几天的相似大盘记忆会合并成一条趋势印象

例如，之前会连续出现：

- `今天一早大家还在念叨：大盘震荡...`
- `今天一早大家还在念叨：大盘走强...`
- `今天一早大家还在念叨：大盘承压...`

现在会收成一条：

- `这几天盘面来回拉扯，大家嘴上一直在念叨大盘；最近一次收在 152.27，单日 +0.28%。`

## 9. 教学上的意义

这一轮调优说明了一件对本科生和研究生都很重要的事：

> 智能体是否“像人”，不只取决于模型强弱，更取决于任务结构、口吻约束、上下文喂法、输出清洗和长期状态管理。

也就是说，真实的 agent 工程不是“换一个更大的模型”，而是：

- 控制提示词结构
- 控制输入信息
- 控制输出边界
- 控制跨轮记忆
- 控制不同角色在不同场景里的表达差异

## 10. 当前边界

虽然这一轮已经明显改善，但仍有两个边界：

1. 旧历史日志不会全部自动变成人话  
   只有可迁移的摘要和记忆会被清洗。
2. 环境摘要和事件摘要仍有少量模板味  
   它们在 [app/engine/game_engine.py](../app/engine/game_engine.py) 里，不完全由 LLM 生成。

## 11. 结论

这轮 prompt 调优的结果，不是“模型更聪明了”，而是：

- 角色更稳定
- 说话更短
- 公开/私下口吻更分明
- 内部代号不再泄露
- 长期记忆更像“挂念”，不再像流水账

因此它真正提升的是：

- 可读性
- 角色辨识度
- 演示效果
- 教学解释性
